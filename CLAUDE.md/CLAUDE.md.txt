# CLAUDE.md — worklog-widget

## 프로젝트 메모리
이 파일은 이 프로젝트에서 Claude Code가 항상 지켜야 할 규칙입니다.

## 사용자 정보
- 3년차 모바일 게임 기획자 (프렌즈팝콘 담당)
- 커뮤니케이션 언어: **한국어**
- 선호 스타일: 간결하고 이해하기 쉽게, 요청 시 상세하게

## 절대 규칙
1. **모르면 질문** — 덜 이해한 상태로 코드 작성 금지
2. **전체 파일 전달** — 수정 시에도 전체 파일 기준으로 응답
3. **외부 라이브러리 금지** — Vanilla HTML/CSS/JS만 사용
4. **한국어 응답**

## 응답 포맷
매 응답 상단에 체크리스트 표시:
```
## 📋 요구사항 체크
- ✅ 완료한 항목
- 🔄 진행 중 항목
- ❌ 미완료/막힌 항목
```

## 코드 컨벤션
- 들여쓰기: 공백 2칸
- CSS: CSS 변수(`--bg`, `--text` 등) 사용, 다크모드 대비 구조
- JS: `const`/`let`만 사용 (`var` 금지), 전역 함수 최소화
- 이벤트: `addEventListener` 또는 `onclick` 속성 둘 다 허용
- 저장: `saveState()` 함수 하나로 통일, 모든 변경 후 호출

## 완성 후 체크리스트
구현 완료 시 아래 항목을 전부 직접 확인하고 결과 보고:
- [ ] 월~금 5일치 카드가 오늘 기준 이번 주 날짜로 자동 생성되는가
- [ ] Q1~Q4 4분면에 업무 추가/삭제/체크가 가능한가
- [ ] 상시 업무 D-DAY가 올바르게 계산되는가 (초과/3일/7일 색상)
- [ ] 주간 갱신 버튼이 확인 모달을 띄우는가
- [ ] 주간 갱신 후 상시 업무는 유지되는가
- [ ] 히스토리 탭에서 과거 주간이 보이는가
- [ ] 페이지 새로고침 후 모든 데이터가 복원되는가
- [ ] JSON 백업/복원이 동작하는가
- [ ] Notion 임베드 환경(iframe) 제약 고려됐는가
- [ ] 모바일 폭(768px)에서 레이아웃 깨지지 않는가
